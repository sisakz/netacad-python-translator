<h1>Adding elements to a list: <font face="courier new">append()</font> and <font face="courier new">insert()</font></h1>

<p>A new element may be <i>glued</i> to the end of the existing list:</p>

<codep>list<mark style="background-color:#e6f2ff;">.append(value)</mark>

</codep>

<br>



<p>Such an operation is performed by a method named <codel>append()</codel>. It takes its argument's value and puts it <strong>at the end of the list</strong> which owns the method.</p>

<p>The list's length then increases by one.</p>


<hr>

<p>The <codel>insert()</codel> method is a bit smarter ‒ it can add a new element <strong>at any place in the list</strong>, not only at the end.</p>

<codep>list<mark style="background-color:#e6f2ff;">.insert(location, value)</mark>

</codep>

<br>

<p>It takes two arguments:</p>

<ul>
<li>the first shows the required location of the element to be inserted; note: all the existing elements that occupy locations to the right of the new element (including the one at the indicated position) are shifted to the right, in order to make space for the new element;</li>

<li>the second is the element to be inserted.</li>
</ul>

Look at the code in the editor. See how we use the <codel>append()</codel> and <codel>insert()</codel> methods. Pay attention to what happens after using <codel>insert()</codel>: the former first element is now the second, the second the third, and so on.</p>


<hr>


<p>Add the following snippet after the last line of code in the editor:</p>

<codep class="syntax-color">numbers.insert(1, 333)
</codep>

<br>

<p>Print the final list content to the screen and see what happens. The snippet above inserts <codel>333</codel> into the list, making it the second element. The former second element becomes the third, the third the fourth, and so on.</p>
