<h1>Integers: octal and hexadecimal numbers</h1>


<p>There are two additional conventions in Python that are unknown to the world of mathematics. The first allows us to use numbers in an <strong>octal</strong> representation.</p>

<p>If an integer number is preceded by an <codel>0O</codel> or <codel>0o</codel> prefix (zero-o), it will be treated as an octal value. This means that the number must contain digits taken from the [0..7] range only.</p>

<p><codel>0o123</codel> is an <strong>octal</strong> number with a (decimal) value equal to <codel>83</codel>.</p>


<p>The <codel>print()</codel> function does the conversion automatically. Try this:</p>

<codep class="syntax-color">print(0o123)
</codep>

<br>


<p>The second convention allows us to use <strong>hexadecimal</strong> numbers. Such numbers should be preceded by the prefix <codel>0x</codel> or <codel>0X</codel> (zero-x).</p>


<p><codel>0x123</codel> is a <strong>hexadecimal</strong> number with a (decimal) value equal to <codel>291</codel>. The print() function can manage these values too. Try this:</p>

<codep class="syntax-color">print(0x123)
</codep>
