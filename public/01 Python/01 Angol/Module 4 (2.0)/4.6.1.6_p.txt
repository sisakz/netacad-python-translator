<h1>How to use a dictionary: the <font face="courier new">keys()</font></h1>

<p>Can dictionaries be <strong>browsed</strong> using the <codel>for</codel> loop, like lists or tuples?</p>

<p>No and yes.</p>

<p>No, because a dictionary is <strong>not a sequence type</strong> − the <codel>for</codel> loop is useless with it.</p>

<p>Yes, because there are simple and very effective tools that can <strong>adapt any dictionary to the <codel>for</codel> loop requirements</strong> (in other words, building an intermediate link between the dictionary and a temporary sequence entity).</p>

<p>The first of them is a method named <codel>keys()</codel>, possessed by each dictionary. The method <strong>returns an iterable object consisting of all the keys gathered within the dictionary</strong>. Having a group of keys enables you to access the whole dictionary in an easy and handy way.</p>


<p>Just like here:</p>

<codep>dictionary = {"cat": "chat", "dog": "chien", "horse": "cheval"}

for key in <mark style="background-color:#e6f2ff;">dictionary.keys()</mark>:
    print(key, "->", dictionary[key]

</codep>

<br>

<p>The code's output looks as follows:</p>

<codep>horse -> cheval
dog -> chien
cat -> chat</codep>
<p style="text-align: right;"><font size="-2"><b>output</b></font></p>

<br>

<h2>The <font face="courier new">sorted()</font> function</h2>

<p>Do you want it <strong>sorted</strong>? Just enrich the <codel>for</codel> loop to get such a form:</p>

<codep>for key in <mark style="background-color:#e6f2ff;">sorted(dictionary.keys())</mark>:

</codep>
<br>

<p>The <codel>sorted()</codel> function will do its best − the output will look like this:</p>

<codep>cat -> chat
dog -> chien
horse -> cheval</codep>
<p style="text-align: right;"><font size="-2"><b>output</b></font></p>
