<h1>Python, más néven CPython</h1>

<p>A Python 2 és a Python 3 mindegyikének több változata is létezik.</p>

<img src="/uploads/media/default/0001/01/c5fcf01b0078c8c99d004c969424bfae3acf67eb.png" width="40%" alt="Python Software Foundation logo" style="float:left">

Először is ott vannak azok a Pythonok, amelyeket a PSF (<a href="https://www.python.org/psf-landing/" target="_blank">Python Software Foundation</a>) köré tömörült emberek tartanak fenn. Ez egy olyan közösség, amelynek célja a Python és környezetének fejlesztése, javítása, bővítése és népszerűsítése. A PSF elnöke maga Guido von Rossum. Ezeket a Pythonokat nevezzük <strong>kanonikusnak</strong>. Ezek egyben <strong>referencia Pythonoknak</strong> is tekinthetők, mivel a nyelv bármely más implementációjának követnie kell a PSF által meghatározott összes szabványt.


<div style="clear: both;"></div>
<br>

<p>Guido van Rossum a "C" programozási nyelvet használta a Python legelső verziójának megvalósításához, és ez a döntés azóta is meghatározó. A PSF-től származó összes Python "C" nyelven készül. Ennek a megközelítésnek számos oka és számos következménye van. Az egyik (talán a legfontosabb) az, hogy ennek köszönhetően a Python könnyen portolható és áttelepíthető minden olyan platformra, amely képes "C" nyelvű programok fordítására és futtatására. (Gyakorlatilag minden platform rendelkezik ezzel a funkcióval, ami számos bővítési lehetőséget nyit meg a Python számára.)</p>

<p>Éppen ezért a PSF implementációját gyakran <strong>CPython</strong> néven emlegetik, ami a legbefolyásosabb Python a világ összes Pythonja közül.</p>

<h1>Cython</h1>

<p>A Python család másik tagja a <strong>Cython</strong>.</p>

<img src="/uploads/media/default/0001/01/d0938abad33db7e330d86e72cd5ddcc75f711bc0.png" width="40%" alt="Cython logo" style="float:left">

A Cython megoldást kínál a Python legfájdalmasabb tulajdonságára - a hatékonyság hiányára. A nagy és összetett matematikai számítások könnyen kódolhatók Pythonban (sokkal könnyebben, mint a "C" vagy bármely más hagyományos nyelven), de az így kapott kód végrehajtása rendkívül időigényes lehet.


<div style="clear: both;"></div>
<br>

<p>Hogyan egyeztethető össze ez a két ellentmondás? Az egyik megoldás az, hogy a matematikai ötleteidet Python nyelven írod meg, és ha teljesen biztos vagy benne, hogy a kódod helyes és jó eredményeket produkál, akkor lefordíthatod "C"-re. Természetesen a "C" sokkal gyorsabban fog futni, mint a tiszta Python.</p>

<p>Ezt próbálja megvalósítani a Cython - automatikusan lefordítja a Python kódot (tiszta és világos, de nem túl gyors) "C" kóddá (egyelőre bonyolult és beszédes, de azért agilis).</p>
