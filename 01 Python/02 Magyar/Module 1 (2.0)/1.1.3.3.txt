<h1>Jython</h1>

<p>A Python egy másik verziója a <strong>Jython</strong>.</p>

<p>A "J" a "Java" rövidítése. Képzelj el egy Pythont, amelyet C helyett Java nyelven írtak. Ez akkor hasznos, ha nagy és összetett, teljes egészében Java nyelven írt rendszereket fejlesztünk, és szeretnénk némi Python rugalmasságot ehhez hozzáadni. A hagyományos CPython nehezen integrálható egy ilyen környezetbe, mivel a C és a Java teljesen más világban él, és nem sok közös megoldásuk van.</p>

<p>A Jython hatékonyabban tud kommunikálni a meglévő Java infrastruktúrával. Ez az oka annak, hogy bizonyos projektek használhatónak és szükségesnek találják.</p>

<p>Megjegyzés: a jelenlegi Jython implementáció a Python 2 szabványokat követi. Egyelőre nincs Python 3-nak megfelelő Jython.</p>

<h1>PyPy és RPython</h1>

<p>Nézd meg az alábbi logót. Ez egy rébusz. Meg tudod oldani?</p>

<img src="/uploads/media/default/0001/01/1bf7a3808081ce1f05a7f7e8d24b024ea904218a.png" width="50%" alt="PyPy logo" style="float:left">

Ez a <strong>PyPy</strong> logója egy Python a Pythonban. A PyPy egy Python-környezet, amit egy Python-szerű nyelven, <strong>RPython</strong> (Restricted Python) írtak. Ez valójában a Python egy részhalmaza.

<br><br>A PyPy mindig csak az épp futtatandó kódrészletet fordítja le.

<div style="clear: both;"></div>
<br>

<p>Ez akkor lehet hasznos, ha valamilyen új funkciót akarsz tesztelni, amelyet talán (de nem feltétlenül) bevezetnek a Python mainstream implementációjába. Ekkor azt könnyebb ellenőrizni a PyPy-vel, mint a CPythonnal, és ezért a PyPy inkább a Pythont fejlesztők eszköze, mint a többi felhasználóé.</p>

<p>Ettől persze a PyPy nem lesz kevésbé fontos vagy kevésbé komoly, mint a CPython.</p>

<p>Ráadásul a PyPy kompatibilis a Python 3 nyelvvel.</p>

<p>A világon sokkal többféle Python létezik. Megtalálod őket, ha rákeresel, de <strong>ez a tanfolyam a CPythonra koncentrál</strong>.</p>
