<h1>Questions and answers</h1>

<p>A programmer writes a program and <strong>the program asks questions</strong>.</p>

<p>A computer executes the program and <strong>provides the answers</strong>. The program must be able to <strong>react according to the received answers</strong>.</p>

<p>Fortunately, computers know only two kinds of answers:</p>

<ul>
<li>yes, this is true;</li>
<li>no, this is false.</li>
</ul>
 

<p>You will never get a response like <i>Let me think....</i>, <i>I don't know</i>, or <i>Probably yes, but I don't know for sure</i>.</p>

 
<p><strong>To ask questions, Python uses a set of very special operators</strong>. Let's go through them one after another, illustrating their effects on some simple examples.</p>

<br>

<h2>Comparison: equality operator</h2>


<p>Question: <strong>are two values equal</strong>?</p>

<p>To ask this question, you use the <codel>==</codel> (equal equal) operator.</p>

<p>Don't forget this important distinction:</p>

<ul>
<li><codel><mark style="background-color:#e6f2ff;">=</mark></codel> is an <strong>assignment operator</strong>, e.g., <codel>a = b</codel> assigns <codel>a</codel> with the value of <codel>b</codel>;</li>
<li><codel><mark style="background-color:#e6f2ff;">==</mark></codel> is the question <i>are these values equal?</i> so <codel>a == b</codel> <strong>compares</strong> <codel>a</codel> and <codel>b</codel>.</li>
</ul>

<p>It is a <strong>binary operator with left-sided binding</strong>. It needs two arguments and <strong>checks if they are equal</strong>.</p>
