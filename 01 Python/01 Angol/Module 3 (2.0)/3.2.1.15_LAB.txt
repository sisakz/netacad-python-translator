<h2>Scenario</h2>

<p>In 1937, a German mathematician named Lothar Collatz formulated an intriguing hypothesis (it still remains unproven) which can be described in the following way:</p>

<ol>
<li>take any non-negative and non-zero integer number and name it <codel>c0</codel>;</li>
<li>if it's even, evaluate a new <codel>c0</codel> as <codel>c0 ÷ 2</codel>;</li>
<li>otherwise, if it's odd, evaluate a new <codel>c0</codel> as <codel>3 × c0 + 1</codel>;</li>
<li>if <codel>c0 ≠ 1</codel>, go back to point 2.</li>
</ol>

<p>The hypothesis says that regardless of the initial value of <codel>c0</codel>, it will always go to 1.</p>

<p>Of course, it's an extremely complex task to use a computer in order to prove the hypothesis for any natural number (it may even require artificial intelligence), but you can use Python to check some individual numbers. Maybe you'll even find the one which would disprove the hypothesis.</p>

<br>

<p>Write a program which reads one natural number and executes the above steps as long as <codel>c0</codel> remains different from 1. We also want you to count the steps needed to achieve the goal. Your code should output all the intermediate values of <codel>c0</codel>, too.</p>

<p>Hint: the most important part of the problem is how to transform Collatz's idea into a <codel>while</codel> loop – this is the key to success.</p>

<p>Test your code using the data we've provided.</p>

c0 = int(input("Enter c0: "))

if c0 > 1:
	steps = 0
	while c0 != 1:
		if c0 %2 != 0:
			cnew = 3 * c0 + 1
		else:
			cnew = c0 // 2
		print(c0)
		c0 = cnew
		steps += 1
	print("steps =",steps)
else:
	print("Bad c0 value")
